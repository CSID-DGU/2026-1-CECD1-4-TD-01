사용자 확인 결과, 오늘 작업의 1순위는 실제 사용 중 종료되는 문제 해결이다.

1. 첨부 전송 크래시
   - 음성 녹음 파일이 첨부된 상태로 모델에 넘기면 앱이 바로 종료된다.
   - 이미지 첨부 후 전송해도 앱이 바로 종료된다.
   - 우선 Kotlin 예외 처리, 파일 정규화 경로, LiteRT-LM 멀티모달 호출 경로를 확인한다.
   - 필요하면 첨부가 있는 요청은 안전하게 텍스트 fallback으로 보내고, 앱 종료를 막는 것을 먼저 처리한다.

2. 상단 UI 공간 과다 사용
   - 현재 `온디바이스 상담` 상단 영역이 너무 커서 대화 목록이 적게 보인다.
   - GPT 스타일처럼 대화 화면은 최소한의 상단 바만 남기고, 자주 누르지 않는 기능은 설정 화면으로 옮긴다.
   - 설정 화면으로 이동할 항목:
     - 테마 설정
     - 대화 세션 기록/관리
     - 모델 정보
     - 건강 정보 포함 여부
     - 건강 정보 기간 선택
     - 사고 모드 설정

3. 키보드 표시 시 레이아웃 문제
   - 입력 키보드를 올렸을 때 남는 공간과 입력창 위치가 아직 어색하다.
   - `windowSoftInputMode`, `imePadding`, 직접 계산한 IME inset 조합을 다시 점검한다.

## 오늘 처리한 내용

- 이미지/음성 첨부 전송 시 앱이 바로 종료되는 문제를 우선 차단했다.
  - 원인 후보는 `Content.ImageFile(...)`, `Content.AudioFile(...)`을 통한 LiteRT-LM 네이티브 멀티모달 호출이다.
  - Kotlin 예외로 잡히기 전에 네이티브 레벨에서 프로세스가 종료될 수 있으므로, 현재는 첨부 파일을 직접 모델에 넘기지 않는 안전 모드로 변경했다.
  - 첨부가 있으면 프롬프트에 "첨부 안전 모드" 안내를 넣고, 사용자가 텍스트로 설명한 내용만 근거로 답하게 한다.
  - 이 처리는 기능 완성보다 앱 종료 방지가 우선인 임시 안정화 조치다.

- 채팅 화면 상단 UI를 줄였다.
  - 기존 모델 정보/건강 상태/메뉴 영역은 화면에서 숨기고, 작은 상태 줄과 `설정` 버튼만 남겼다.
  - 설정 다이얼로그에 모델 선택, 프롬프트, 기억, 세션 내보내기/불러오기, 건강 포함/기간, 사고 모드, 테마 선택을 모았다.

- 키보드 표시 시 하단 공간 문제를 완화했다.
  - 입력창에서 `navigationBarsPadding()` 중복 적용을 제거하고, 기존 IME inset 기반 하단 패딩만 사용하도록 조정했다.

- 빌드 설정
