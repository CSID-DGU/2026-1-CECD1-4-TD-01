# 2026-05-27 작업 기록

## 현재 목표

Android 상담 앱에서 온디바이스 Gemma 4 E4B IT 모델을 실행하고, 사용자의 사진/헬스 데이터/대화 기록을 상담 맥락으로 활용할 수 있는 구조를 만든다.

## 모델 실행

- GGUF/llama.cpp 경로는 실제 기기에서 안정성이 낮아 UI에서 숨기는 방향으로 정리했다.
- Google AI Edge의 현재 권장 경로에 맞춰 LiteRT-LM을 기본 런타임으로 전환했다.
- 사용 모델:
  - `models/gemma-4-E4B-it.litertlm`
  - 약 3.66GB
- 앱에서는 사용자가 `모델 로드` 버튼으로 `.litertlm` 파일을 선택한다.
- 선택된 모델은 앱 전용 저장소로 복사한 뒤 LiteRT-LM `Engine`에 파일 경로로 전달한다.
- 복사 중 실패한 모델 파일이 남지 않도록 `.partial` 파일에 먼저 복사한 뒤 최종 파일로 교체하는 방식으로 바꿨다.

## Gemma 프롬프트 방식

- Gemma 공식 문서 기준으로 별도 system role보다는 첫 user prompt에 시스템 수준 지시를 포함하는 방식이 권장된다.
- 따라서 LiteRT-LM `ConversationConfig(systemInstruction = ...)` 방식 대신, 첫 사용자 메시지에 상담 지침을 함께 포함하도록 엔진을 수정했다.
- 기본 상담 지침에는 다음을 포함했다.
  - 감정 반영
  - 판단/단정 회피
  - 한 번에 하나씩 명확한 질문
  - 사실/감정/욕구/위험 신호 구분
  - 실행 가능한 작은 단계 제안
  - 자살/자해/타해/학대/폭력/응급 위험 신호 우선 대응
  - 한국: 109, 1577-0199, 112, 119 안내
  - 미국: 988 안내
- 앱 안에서 시스템 프롬프트를 수정할 수 있는 `프롬프트` 버튼을 추가했다.

## 채팅 UX

- LiteRT-LM `sendMessageAsync`를 사용해 스트리밍 응답을 표시하도록 바꿨다.
- 답변 말풍선이 먼저 생성되고, 생성되는 텍스트가 들어오는 대로 갱신된다.
- `**굵게**` 형태의 간단한 Markdown bold 렌더링을 추가했다.
- 키보드가 열릴 때 큰 빈 여백이 생기는 문제를 줄이기 위해 중복 IME 패딩을 제거했다.
- `AndroidManifest.xml`의 `windowSoftInputMode="adjustResize"`는 유지했다.

## 갤러리

- Android 14+의 부분 사진 접근 권한을 고려해 상태를 구분한다.
  - 권한 없음
  - 일부 사진 접근
  - 전체 갤러리 접근
- 권한이 없거나 일부 접근일 때 앱 내부 안내창을 먼저 보여준다.
- 전체 갤러리 접근은 사용자에게 명시적으로 요청하는 흐름으로 유지한다.
- Android가 권한창을 반복해서 띄워주지 않는 경우가 있으므로, 최종적으로는 앱 설정 화면으로 보내는 버튼도 유지한다.
- 향후 UX 방향:
  - 기본: `사진 추가`는 Android Photo Picker 기반으로 권한 없이 사용
  - 선택: `전체 갤러리 보기`는 명시적 권한 요청
  - 설정: 권한 변경은 앱 설정으로 이동

## 헬스 데이터

- Health Connect를 사용한다.
- 현재 읽는 항목:
  - 걸음 수
  - 총 소모 칼로리
  - 활동 칼로리
  - 이동 거리
  - 평균 심박수
  - 수면 시간
- Samsung Health와 Health Connect 연동을 검토했다.
  - Galaxy Fit3는 수면 추적을 지원한다.
  - Samsung Health는 Health Connect로 걸음, 운동, 심박, 수면 등을 동기화할 수 있다.
  - 하지만 앱에서 Samsung Health에게 “지금 Health Connect에 써라”를 강제하는 API는 없다.
  - 사용자는 Samsung Health에서 `설정 > Samsung 계정과 동기화 > 지금 동기화`를 누르거나, Samsung Health 홈 화면 새로고침으로 동기화를 유도할 수 있다.
  - 오늘 처음 Health Connect를 연결했다면 과거 수면 데이터가 바로 들어오지 않을 수 있다.

## 다음에 할 일

1. 현재 작업 트리 빌드 상태 확인
   - `.\gradlew.bat :app:assembleDebug`
   - 실패하면 이미지/음성/세션 입출력 중간 구현부터 정리

2. 이미지/음성 입력 완성
   - 이미지 선택 버튼
   - 음성 인식 버튼
   - 음성 파일 첨부 버튼
   - 메시지 말풍선에 첨부 상태 표시
   - LiteRT-LM 멀티모달 모델이 실제로 이미지/오디오 입력을 받는지 기기 테스트

3. 세션 저장/복원 완성
   - JSON export/import
   - 시스템 프롬프트 포함
   - 첨부 파일은 경로만 저장할지, 별도 복사본까지 묶을지 결정

4. 헬스 데이터를 프롬프트에 자동 주입
   - 최근 7일 또는 30일 요약을 생성
   - 걸음, 활동량, 수면, 심박, 칼로리, 거리 등을 상담 맥락으로 전달
   - 사용자가 원할 때만 포함하는 토글 필요
   - 모델에게 원자료를 그대로 던지기보다 요약 문장으로 전달하는 편이 안전함

5. WHO 권고사항 기반 생활 패턴 개선 안내
   - 신체활동, 수면, 정신건강 관련 WHO 또는 공신력 있는 가이드라인을 확인
   - 앱 프롬프트에 “진단이 아닌 생활습관 지원”으로 반영
   - 위험 신호가 있으면 생활습관 조언보다 위기 대응을 우선
