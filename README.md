# Counseling — On-Mom

**Counseling(On-Mom)**은 청소년 사용자가 상담을 시작하기 어려운 순간에, 스마트폰 안의 일상 데이터에서 추출한 생활 맥락을 바탕으로 자연스러운 첫 질문과 상담 응답을 보조하는 **온디바이스 AI 상담 보조 앱**입니다.

사용자의 대화, 갤러리 이미지, 건강 데이터, 통화 기록, 앱 사용 패턴, 음성 감정 단서를 기기 내부에서 분석하고, 그 결과를 하나의 상담 브리프로 정리해 온디바이스 LLM 상담 응답에 반영합니다.

> 이 앱은 의료 진단이나 치료를 대신하지 않습니다.  
> 사용자가 자신의 감정과 생활 흐름을 안전하게 돌아보고, 필요한 경우 전문적인 도움으로 연결될 수 있도록 돕는 상담 보조 도구입니다.

---

## 프로젝트 핵심 요약

On-Mom은 단순히 사용자가 입력한 텍스트만 보고 답하는 챗봇이 아니라, 사용자가 허용한 데이터 안에서 **생활 맥락을 요약하고 상담 흐름을 보조하는 앱**입니다.

| 구분 | 내용 |
| --- | --- |
| 대상 | 상담 진입에 부담을 느끼는 청소년 사용자 |
| 목표 | 비대면·온디바이스 환경에서 생활 맥락 기반 상담 보조 제공 |
| 핵심 데이터 | 대화, 갤러리 이미지, 건강 데이터, 통화 기록, 앱 사용 패턴, 음성 |
| 핵심 방식 | 기기 내부 분석 → 상담 브리프 생성 → 온디바이스 LLM 응답 |
| 안전 원칙 | 진단·단정 금지, 사용자 확인 우선, 위험 신호 시 안전 확인 우선 |
| 개인정보 원칙 | 원본 민감 데이터 외부 전송 없이 로컬 처리 중심 설계 |

## 개발 배경

기존 청소년 상담은 학교 상담실, 위클래스, 대면 상담처럼 물리적인 공간과 시간에 영향을 받는 경우가 많습니다. 또한 상담을 받으러 간다는 사실 자체가 부담이나 낙인으로 느껴질 수 있습니다.

반면 많은 사용자는 이미 AI 기반 대화 서비스를 고민 상담의 접점으로 활용하고 있습니다. 하지만 일반적인 AI 챗봇은 사용자의 생활 맥락을 충분히 알지 못한 채 텍스트만 보고 응답하는 한계가 있습니다.

On-Mom은 이 문제를 줄이기 위해 다음 방향을 선택했습니다.

- 상담실에 가기 전에도 접근 가능한 모바일 상담 보조 도구
- 민감한 데이터를 서버로 보내지 않는 온디바이스 처리
- 텍스트 대화뿐 아니라 생활 데이터, 갤러리, 음성 감정 단서를 함께 활용
- 상담자가 아닌 사용자의 자기 이해와 다음 행동 선택을 돕는 구조
- 위험 신호가 있을 때 일반 조언보다 안전 확인을 우선하는 응답 정책

## 핵심 아이디어

### 1. 온디바이스 상담

상담 맥락은 매우 민감한 정보입니다. On-Mom은 원본 사진, 음성, 통화 기록, 앱 사용 기록, 건강 데이터를 외부 서버에 직접 전달하지 않고, 가능한 한 기기 내부에서 분석합니다.

LLM에는 원본 데이터가 아니라 상담에 필요한 요약 정보만 전달하는 구조를 지향합니다.

### 2. 멀티모달 생활 맥락 분석

사용자는 자신의 어려움을 처음부터 명확히 설명하기 어렵습니다. On-Mom은 사용자가 허용한 데이터를 바탕으로 다음과 같은 보조 맥락을 구성합니다.

- 갤러리 이미지 기반 생활 장면, 활동, 관심사 요약
- 건강 데이터 기반 활동량, 거리, 칼로리, 심박, 수면 리듬 요약
- 통화 기록과 앱 사용 패턴 기반 사회적 연결, 고립 신호, 모바일 과사용 흐름 요약
- 음성 입력 기반 현재 발화의 정서 단서 요약
- 중요 기억 기반 반복적으로 참고할 개인 맥락 유지

### 3. 상담 브리프 기반 응답

각 분석 모듈은 독립적으로 결과를 만들고, 앱은 이를 상담용 브리프로 통합합니다.

상담 브리프는 진단 결과가 아니라, 상담을 어떤 주제로 시작하면 좋을지 정리하는 보조 자료입니다. 예를 들어 “사진을 보니 우울해 보여요”처럼 단정하지 않고, “요즘 생활 리듬이 조금 달라진 것 같은데, 최근에 가장 피곤했던 순간이 있었나요?”처럼 확인형 질문을 만들기 위한 참고 정보로 사용합니다.

### 4. 사용자 통제

사용자는 상담 응답에 어떤 데이터가 들어갈지 직접 조절할 수 있습니다.

- 갤러리 분석 제외
- 건강 데이터 제외
- 생활 패턴 제외
- 분석 기간 설정
- 오디오 직접 분석 켜기/끄기
- 중요 기억 삭제
- 세션 내보내기/불러오기
- 모델 파일 선택
- 테마와 글자 크기 조정

### 5. 안전 우선

On-Mom은 상담형 응답을 제공하지만, 위험 상황에서는 일반적인 공감이나 생활 조언보다 안전 확인을 먼저 수행합니다.

- 자해, 자살, 타인 위해, 폭력, 학대 등 위험 신호 확인
- 이미지나 생활 데이터만으로 위험 상황 확정 금지
- 사용자의 직접 발화와 현재 안전 여부 확인 우선
- 필요한 경우 보호자, 전문기관, 긴급 서비스 연결 안내

## 전체 사용 흐름

1. 사용자는 앱을 실행하고 필요한 권한을 직접 허용합니다.
2. 상담 탭에서 현재 고민이나 감정을 입력합니다.
3. 앱은 사용자가 허용한 데이터만 바탕으로 상담 맥락을 구성합니다.
4. 갤러리 분석은 최근 이미지에서 생활 맥락과 안전 보조 신호를 요약합니다.
5. 건강 데이터는 활동량, 칼로리, 거리, 심박, 수면 같은 생활 리듬을 정리합니다.
6. 생활 패턴은 통화 기록과 앱 사용 정보를 바탕으로 사회적 연결, 고립 신호, 과사용 흐름을 요약합니다.
7. 음성 분석은 발화 텍스트와 음향 단서를 바탕으로 현재 감정 흐름을 보조적으로 반영합니다.
8. 중요 기억은 상담에서 반복적으로 참고할 개인 맥락을 저장하고 관리합니다.
9. 프롬프트 생성기는 분석 결과를 하나의 상담 브리프로 통합합니다.
10. 온디바이스 LLM은 이 브리프를 진단 근거가 아니라 대화 보조 자료로 사용해 응답합니다.

## 시스템 아키텍처

```text
사용자 입력 / 권한 기반 데이터
│
├─ 상담 대화
│  └─ 현재 고민, 감정, 사용자의 직접 발화
│
├─ 갤러리 이미지
│  └─ ImageFilter → EventGrouping → ML Kit → Safety Overlay → Gallery Brief
│
├─ 건강 데이터
│  └─ Health Connect → 활동량/거리/칼로리/심박/수면 요약 → Health Brief
│
├─ 생활 패턴
│  └─ CallLog / UsageStats → 사회적 연결/고립/과사용/일간 리듬 요약 → Phenotype Brief
│
├─ 음성 입력
│  └─ STT / 음향 특성 / 감정 분류 → Emotion Brief
│
└─ 중요 기억
   └─ 사용자 선호, 상담 원칙, 반복 참고 맥락

        ↓

RAG-lite / Prompt Builder
- 상담 원칙
- 현재 대화
- 사용자 기억
- 건강 요약
- 생활 패턴 요약
- 갤러리 분석 요약
- 음성 감정 요약
- 안전 정책

        ↓

On-device LLM
Gemma 4 + LiteRT-LM

        ↓

상담형 응답
- 감정 반영
- 상황 정리
- 확인형 질문
- 작은 다음 행동
- 위험 신호 시 안전 확인
```

## 주요 기능

## 1. 상담

상담 화면은 사용자가 고민을 입력하고 온디바이스 상담 모델의 응답을 받는 중심 화면입니다. 사용자는 하단 입력창에서 메시지를 작성하고, 하단 내비게이션을 통해 갤러리, 건강, 생활 패턴, 설정 화면으로 이동할 수 있습니다.

<img src="종설_wkfy/결과%20사진/대화.jpg" width="320" alt="상담 대화 화면">

상담 응답은 다음 원칙을 따릅니다.

- 사용자의 감정과 상황을 먼저 반영합니다.
- 바로 해결책을 단정하지 않고 선택 가능한 관점을 제시합니다.
- 공부, 생활, 관계, 건강 등 사용자가 말한 주제를 중심으로 질문 방향을 잡습니다.
- 필요하면 한 번에 하나의 명확한 질문을 합니다.
- 위험 신호가 있으면 현재 안전 여부를 먼저 확인합니다.
- 의료, 법률, 재정 같은 전문 영역은 전문가 상담을 권합니다.

---

## 2. 갤러리 분석

갤러리 분석은 사용자가 허용한 이미지에서 생활 흐름을 요약하는 기능입니다. 이미지를 단순히 분류하는 것이 아니라, 상담에서 참고할 수 있는 생활 맥락을 문서 형태로 정리합니다.

<img src="종설_wkfy/결과%20사진/갤러리1.jpg" width="320" alt="갤러리 분석 화면">

분석 결과 화면에서는 상담에 바로 사용할 수 있는 요약을 확인할 수 있습니다.

<img src="종설_wkfy/결과%20사진/갤러리2.jpg" width="320" alt="갤러리 분석 결과 화면">

갤러리 분석 결과에는 다음 정보가 포함됩니다.

- 상담 진행 전략
- 안전 및 폭력 보조 확인
- 현재 주요 생활 맥락
- 지금 물어볼 핵심 질문
- 이후 활용 가능한 회복 자원 후보
- 상담 후반부에 제안할 수 있는 작은 행동 후보

### 갤러리 분석 파이프라인

```text
ImageFilter
    ↓
EventGrouping
    ↓
ML Kit Image Labeling
    ↓
ImageAnalyzer
    ↓
Safety Overlay
    ↓
Counseling Prompt
```

### ImageFilter

- MediaStore를 통해 이미지 이름, 경로, 수정 시간을 수집합니다.
- Screenshots, Download, KakaoTalk 등 상담 맥락과 직접 관련이 낮은 폴더는 제외할 수 있습니다.
- 실제 촬영 기반 생활 사진을 중심으로 분석 대상을 구성합니다.

### EventGrouping

- 짧은 시간 안에 연속 촬영된 유사 사진을 하나의 activity event로 묶습니다.
- 기본 기준은 5분 이내 촬영과 유사 해시입니다.
- 같은 장면을 여러 장 찍었을 때 활동량이나 선호도가 과대 반영되는 문제를 줄입니다.
- 이후 분석은 단순 photo count가 아니라 event count 중심으로 수행합니다.

### ML Kit Image Labeling

- Android 기기 내부에서 이미지 라벨을 추출합니다.
- 음식, 사람, 책상, 운동, 실내, 반려동물 등 일반 생활 맥락을 파악하는 데 사용합니다.
- 초기 개발 단계에서 직접 CNN을 학습하기보다 안정적인 온디바이스 사전 모델을 활용했습니다.

### Safety Overlay

- 상처, 피, 폭력 가능성 같은 단서는 일반 생활 패턴 분석과 분리합니다.
- 이미지 단독으로 자해, 폭력, 성폭력 피해를 확정하지 않습니다.
- 위험 가능성이 있더라도 단정 대신 현재 안전 여부를 확인하는 질문 생성을 우선합니다.

## 3. 건강 데이터

건강 화면은 Health Connect 기반 생활 리듬 요약 화면입니다. 사용자는 권한을 연결하고 최신 데이터를 새로고침할 수 있으며, 주간 또는 월간 단위로 활동 정보를 확인할 수 있습니다.

<img src="종설_wkfy/결과%20사진/헬스1.jpg" width="320" alt="건강 데이터 요약 화면">

아래로 스크롤하면 수면 시간까지 포함한 상세 요약을 확인할 수 있습니다.

<img src="종설_wkfy/결과%20사진/헬스2.jpg" width="320" alt="건강 데이터 상세 화면">

건강 화면에서 다루는 정보는 다음과 같습니다.

- 걸음 수
- 총 소모 칼로리
- 활동 칼로리
- 이동 거리
- 평균 심박수
- 수면 시간

이 데이터는 “최근 수면이 줄었으니 우울하다”처럼 단정적으로 사용하지 않습니다. 대신 사용자가 피로, 긴장, 무기력, 생활 리듬 변화를 이야기할 때 조심스럽게 참고하는 보조 맥락으로 사용합니다.

---

## 4. 생활 패턴 / 디지털 피노타입

생활 패턴 화면은 통화 기록과 앱 사용 정보를 바탕으로 최근 생활 흐름을 요약합니다. 사회적 연결, 고립 신호, 모바일 과사용, 일간 리듬을 한눈에 보여주고, 통화 패턴을 지난주와 비교합니다.

<img src="종설_wkfy/결과%20사진/피노타입.jpg" width="320" alt="생활 패턴 화면">

주요 지표는 다음과 같습니다.

| 지표 | 설명 |
| --- | --- |
| 사회적 연결 | 주간 통화량, 고유 연락처 수, 연락 빈도 |
| 고립 신호 | 일정 기간 연락 감소 여부 |
| 모바일 과사용 | 일평균 앱 사용 시간 |
| 일간 리듬 | 하루 시간대별 앱 사용 흐름 |
| 통화 패턴 | 이번 주와 지난주의 통화량 비교 |
| 활동 흐름 | 걸음 수, 수면, 심박 등 건강 데이터와 함께 참고 |

생활 패턴은 상담 중 사용자의 생활 변화를 물어볼지 판단하는 보조 신호입니다. 고립, 과사용, 리듬 변화 같은 표현은 진단명이 아니라 대화에서 다루는 참고 지표입니다.

## 5. 음성 감정 분석

음성 분석은 사용자의 발화에서 텍스트만으로는 알기 어려운 정서 단서를 참고하기 위한 기능입니다.

같은 “괜찮아요”라는 문장도 목소리의 높낮이, 속도, 떨림, 에너지에 따라 전혀 다른 의미가 될 수 있습니다. On-Mom은 음성 감정 분석 결과를 상담 응답의 보조 맥락으로 반영합니다.

### 음성 분석 흐름

```text
음성 입력
    ↓
16kHz 음성 전처리
    ↓
STT / 음향 특성 추출
    ↓
Wav2Vec2 또는 MobileNet-v2 계열 감정 분류
    ↓
감정 label + confidence score
    ↓
Prompt Generator
    ↓
Gemma 4 상담 응답
```

음성 분석 결과는 사용자의 상태를 확정하는 데 사용하지 않습니다. 현재 발화의 정서 분위기를 조심스럽게 반영하고, 사용자의 말과 충돌할 경우 사용자의 직접 발화를 우선합니다.

## 상담 응답 설계

Counseling의 응답은 한 번에 답을 주는 방식보다 대화를 이어가기 쉬운 구조를 지향합니다.

1. 사용자의 감정과 의도를 짧게 반영합니다.
2. 상황을 판단하거나 진단하지 않습니다.
3. 필요하면 한 번에 하나의 명확한 질문을 합니다.
4. 여러 해결책을 동시에 강요하지 않고 선택지를 줄입니다.
5. 사용자가 바로 실행할 수 있는 작은 행동을 제안합니다.
6. 사용자가 원하지 않는 데이터 해석을 강요하지 않습니다.
7. 위험 신호가 있으면 안전 확인과 긴급 지원 안내를 우선합니다.

### 예시 응답 방향

```text
사용자: 요즘 그냥 아무것도 하기 싫어.

응답 방향:
1. 감정 반영
2. 무기력 단정 금지
3. 최근 가장 버거웠던 순간 하나만 묻기
4. 건강/생활 패턴은 보조 맥락으로만 참고
5. 바로 해결책을 여러 개 제시하지 않기
6. 위험 신호가 있으면 안전 여부 확인
```

## 구현 과정에서 해결한 문제

### 1. 모바일에서 LLM 로드가 불안정한 문제

초기에는 GGUF 기반 실행 구조를 검토했지만, Android/Kotlin 환경에서 안정적인 구동과 모바일 최적화에 한계가 있었습니다. 이후 Google AI Edge 환경에 적합한 LiteRT-LM 기반 구조로 전환해 온디바이스 LLM 구동 가능성을 높였습니다.

### 2. 너무 많은 권한을 요구하는 문제

상담 앱은 갤러리, 통화 기록, 앱 사용량, 건강 데이터처럼 민감한 정보를 다룹니다. 모든 권한을 한 번에 요구하면 사용자가 부담을 느낄 수 있으므로, 기능별 권한 흐름을 분리했습니다.

사용자는 필요한 기능만 허용할 수 있고, 설정에서 특정 데이터 유형을 상담 맥락에서 제외할 수 있습니다.

### 3. 갤러리 이미지가 너무 많은 문제

모든 이미지를 분석하면 속도와 배터리, 사용성 문제가 발생할 수 있습니다. 따라서 폴더 필터링, 유사 이미지 묶기, event count 기반 분석을 통해 상담 맥락에 필요한 이미지 중심으로 처리하도록 설계했습니다.

### 4. 분석 결과가 진단처럼 보일 수 있는 문제

갤러리, 건강, 생활 패턴, 음성 감정 분석은 모두 오판 가능성이 있습니다. 따라서 분석 결과를 확정 판단이 아니라 상담 질문을 정리하기 위한 보조 정보로 제한했습니다.

### 상담 접근성 향상

대면 상담에 대한 부담이 있거나, 상담실 방문 자체가 부담스러운 사용자가 앱을 통해 먼저 자신의 감정과 생활 흐름을 정리할 수 있습니다.

### 자기 이해 보조

사용자는 수면, 활동량, 통화량, 앱 사용 흐름, 갤러리 생활 장면 등을 통해 최근 자신의 생활 리듬을 돌아볼 수 있습니다.

### 상담 전 사전 정리 도구

학교 상담이나 전문 상담 전에 사용자가 자신의 최근 상태를 설명하기 어려운 경우, 생활 패턴 요약이 초기 상담 방향을 잡는 보조 자료가 될 수 있습니다.

### 온디바이스 AI 활용 가능성

민감한 데이터를 서버로 보내기 어려운 상담, 교육, 헬스케어, 군 장병 상담, 고립 청년 지원 등으로 확장할 수 있습니다.

## 한계와 개선 방향

### 1. 안전 관련 이미지 판단 한계

ML Kit 기본 모델은 일반 생활 이미지 라벨링에는 강점이 있지만, 상처와 자해흔, 폭력 상황을 정확히 구별하는 데는 한계가 있습니다.

개선 방향:

- 위험 상황 시나리오를 더 구체화
- 안전 관련 이미지 데이터셋 추가 검토
- 이미지 단독 판단 금지 원칙 유지
- 현재 안전 여부 확인 질문 강화

### 2. 유사 사진 묶기 한계

현재는 해시 기반 유사도와 시간 기준으로 이벤트를 묶습니다. 하지만 각도, 조명, 구도가 크게 달라지면 같은 상황도 별도 이벤트로 분리될 수 있습니다.

개선 방향:

- MobileCLIP 등 경량 이미지-텍스트 모델 실험
- 모바일 환경에서 가능한 의미 기반 유사도 측정 도입
- 처리 속도와 배터리 사용량 균형 조정

### 3. 음성 감정 분석 정확도

음성 감정은 개인차, 녹음 환경, 소음, 말투에 영향을 크게 받습니다.

개선 방향:

- 한국어 감정 음성 데이터 추가 학습
- 사용자 직접 발화와 감정 모델 결과가 충돌할 때 직접 발화 우선
- 상담 효과 지표로 감정 흐름 변화를 보조적으로만 사용

### 4. 실제 상담 안전성 검증 필요

본 프로젝트는 상담 보조 앱 프로토타입입니다. 실제 서비스 수준으로 배포하려면 상담 전문가 검토, 사용자 테스트, 위기 대응 정책 보강, 임상적 검증이 필요합니다.
